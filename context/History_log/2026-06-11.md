## — SSO callback: handle isTransferable (progressive sign-up/in)

### สิ่งที่ทำ
- เขียน `/sign-up/sso-callback` ใหม่ตาม Clerk's official Future API pattern:
  - `signUp.status === "complete"` → finalize signUp + migrate
  - `signUp.isTransferable` → `signIn.create({transfer:true})` → finalize signIn (กรณี email ซ้ำกับ user เดิม)
  - `existingSession` → `clerk.setActive()`
  - fallback → `/sign-up`
- เขียน `/sign-in/sso-callback` ใหม่แบบ symmetric:
  - `signIn.status === "complete"` → finalize signIn
  - `signIn.isTransferable` → `signUp.create({transfer:true})` → finalize signUp (กรณี user ใหม่ใช้ Google login ตรงๆ)
  - handle `needs_second_factor` / `needs_new_password` → กลับ /sign-in
  - `existingSession` → setActive
- ใช้ `hasRun = useRef(false)` กัน effect ยิงซ้ำ
- เพิ่ม `<div id="clerk-captcha" />` ใน loading UI (รองรับ transfer ที่ต้อง captcha)
- preserve migration logic: `readPendingHistory`/`clearPendingHistory`/`savePendingHistoryAction`/`claimAnonymousMBTIResult`
- รองรับ `session.currentTask` → redirect `/sign-{in|up}/tasks/${key}`

### Files changed
- `src/app/sign-up/sso-callback/page.tsx` — rewrite ทั้งไฟล์
- `src/app/sign-in/sso-callback/page.tsx` — rewrite ทั้งไฟล์

### หมายเหตุ
- Progressive sign-up ใช้งานได้แล้ว: กด "เข้าด้วย Google" ที่ /sign-in โดยที่ยังไม่มีบัญชี → auto-create
- ฝั่งกลับกัน: register ด้วย Google โดยที่มี user เดิม (email/password) → ลิงก์ OAuth กับบัญชีเดิม
