## 14:05 — Fix sign-in/sign-up button bug (loading state ค้าง)

### สิ่งที่ทำ
- วิเคราะห์ปัญหา "กดปุ่มแล้วเหมือนไม่ทำงาน" บนหน้า `/sign-in` และ `/sign-up`
- Root cause: ใช้ `fetchStatus === "fetching"` จาก `useSignIn()`/`useSignUp()` เป็น `loading` — ค่านี้เป็น `"fetching"` ระหว่าง in-flight call ใดๆ ของ Clerk resource (รวมถึง initial load / revalidate) ทำให้ `SubmitButton` และ `OAuthButtons` ถูก `disabled` โดยไม่เกี่ยวกับ action ที่ user กด — ดูเหมือนปุ่มกดไม่ติด
- แก้โดยเพิ่ม local `submitting` state แยกต่างหาก, set `true`/`false` รอบๆ `handleCredentials`/`handleMFA` (sign-in) และ `handleDetails`/`handleVerify` (sign-up) ด้วย try/finally — ใช้แทน `fetchStatus`
- แก้จุดรอง: sign-in MFA step ใช้ `<Field id="code" name="code">` ซ้ำกัน 2 แบบ (OTP vs backup code) ทำให้ React reuse DOM element เดิมข้าม mode — เพิ่ม `key="otp"` / `key="backup"` ให้ Field เพื่อบังคับ remount ตอนสลับโหมด
- TypeScript noEmit ผ่าน clean

### Files changed
- `src/app/sign-in/[[...sign-in]]/page.tsx` — เปลี่ยนจาก `fetchStatus` เป็น `submitting` state, เพิ่ม `key` ให้ MFA code Field
- `src/app/sign-up/[[...sign-up]]/page.tsx` — เปลี่ยนจาก `fetchStatus` เป็น `submitting` state

### หมายเหตุ
- ยังไม่ได้ reproduce ใน browser จริง — รอ user ทดสอบยืนยันว่าปุ่มกดติดปกติแล้ว
