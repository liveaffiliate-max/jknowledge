## — Compare chart: COTMES label fix + curated palette

### Problem
1. กสพท. (COTMES) ถูกเก็บใน DB เหมือนเป็น `University` ตัวหนึ่ง → ตอน compare เส้น 2 เส้น แสดง legend ว่า "กสพท. | COTMES" ซ้ำกัน ทำให้แยกไม่ออกว่าเป็นมหาลัยไหนจริง
2. ใช้ `university.color` เป็นสีเส้น → หลายมหาลัยมีสีคล้ายกัน (มก./มข./มจธ. แดงเหมือนกัน, KU/มข./ม.บูรพา เขียวคล้ายกัน) → เส้นซ้อนกันแยกไม่ออก

### สิ่งที่ทำ
- สร้าง `src/lib/faculty-label.ts`
  - `isCOTMES(uni)` — detect ด้วย `name.includes("กลุ่มสถาบัน")` หรือ `shortName.startsWith("กสพท")`
  - `getFacultyLabelParts(faculty)` — return `{ primaryShort, primaryName, channel? }`
    - กรณี COTMES: ดึงชื่อมหาลัยจริงจาก `faculty.name` ผ่าน regex map (22 มหาลัยสมาชิก กสพท) → `primaryShort = "จุฬาฯ"`, `primaryName = "คณะทันตแพทยศาสตร์"`, `channel = "กสพท"`
  - `COMPARE_PALETTE` — 4 สี categorical (green-600 / blue-600 / orange-600 / purple-600) แยกได้ชัด color-blind safe ใช้ทั้ง chart และ table
  - `paletteColor(idx)` — pick by slot index
- อัปเดต `compare-trend-chart.tsx`:
  - เปลี่ยน stroke color จาก brand → `paletteColor(idx)`
  - Legend chip + Tooltip ใช้ label parts ใหม่ พร้อมแสดง badge "กสพท" สีฟ้าอ่อนถ้าเป็นรอบ consortium
- อัปเดต `compare-table.tsx` SlotCard:
  - ใช้ label parts เดียวกัน — แสดง "จุฬาฯ" + "คณะทันตแพทยศาสตร์" แทน "กสพท. | COTMES · คณะ...จุฬาฯ"
  - เพิ่ม accent bar 4px ที่ขอบบน card สีตรงกับเส้นในกราฟ → จับคู่ภาพได้ทันที
  - ย้าย Trophy badge "เอื้อมถึง" เข้าใน header (จากเดิม -top-2 ที่ overlap กับ accent bar)

## — Phase 2: Same Major Across Universities

### Goal
หน้าใหม่ `/analyze/compare/major/[slug]` — เลือกคณะ X แล้วเห็นทุกมหาลัยที่เปิดสอน คณะนี้ พร้อมคะแนน เรียงจากเข้าง่ายสุด เป็น SEO play เพราะตอบคำถาม "เรียนคณะ X ที่ไหนคะแนนต่ำสุด"

### สิ่งที่ทำ
- **Canonical major key util** (`src/lib/major-canonical.ts`)
  - `getCanonicalMajorKey({name, majorName})` — exact name + majorName หลัง trim whitespace
  - `majorSlugFromKey(key)` / `parseMajorSlug(slug)` — URL round-trip ด้วย encodeURIComponent (รองรับ Thai)
  - `formatMajorLabel(parts)` — display
  - MVP: exact match — รองรับเคสหลักทั้ง "คณะแพทยศาสตร์" (no major) และ "วิศวกรรมคอมพิวเตอร์" (with major)
  - Known limit: ชื่อคณะ variant ("คณะแพทยศาสตร์ ศิริราช" vs "คณะแพทยศาสตร์") จะแยกกัน — defer ไปยัง synonym map ทีหลัง
- **Server queries** (`src/server/queries.ts`)
  - `getMajorComparison(slug)` — รวมทุก Faculty ที่ name+majorName ตรง dedupe ตาม uni (เลือก row ที่มีคะแนนเยอะสุด) เรียง min asc มี `unstable_cache` revalidate 3600
  - `getPopularMajorSlugs(min=3, limit=60)` — major ที่มี ≥ 3 มหาลัยเปิด สำหรับ sitemap
- **Components**
  - `MajorComparisonChart` + lazy wrapper — Recharts horizontal bar, สีตาม curated palette แต่ละแถบ มี tooltip ระบุ ที่นั่ง+ตำแหน่ง dynamic height ตามจำนวน entries
  - `MajorComparisonTable` — sortable header (min/seats/shortName), mini-bar inline ใน column ต่ำสุด, mobile = compact card stack, แต่ละ row link ไป `/scores/[uniSlug]/[facultyId]`
- **Route** `/analyze/compare/major/[slug]/page.tsx` + `loading.tsx`
  - `generateMetadata` — title/description/OG/canonical ตาม slug
  - Hero มี breadcrumb, label, summary chips (uniCount, total seats, easiest, hardest)
  - JSON-LD `ItemList` ส่งให้ Google rich result
  - notFound() ถ้าไม่พบหรือไม่มี entries
- **Sitemap** — เพิ่ม `/analyze/compare/major/<slug>` ตาม popular majors, priority scale ตาม uniCount (max 0.8), + `/analyze/compare` ใน static routes
- **ResultCard CTA** (`result-card.tsx`) — ปุ่ม "ดูคณะนี้ที่มหาลัยอื่น" link จาก analyze result โดยคำนวณ slug จาก faculty.name + majorName
