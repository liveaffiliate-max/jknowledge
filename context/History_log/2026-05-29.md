## — MBTI Dynamic Quiz: Likert Scale + Adaptive Ordering

### สิ่งที่ทำ

**Approach 1 — Likert Scale (5 ระดับ)**
- แทนที่ binary A/B ด้วย 5 ระดับ: 1=strongly A, 2=lean A, 3=neutral, 4=lean B, 5=strongly B
- Weight: 1→+2, 2→+1, 3→0, 4→−1, 5→−2 (neutral ไม่นับ)
- UI: Option A card บนสุด + 5 scale buttons ตรงกลาง + Option B card ล่าง
- Color: A=green, neutral=gray, B=blue — highlighted card เปลี่ยนตาม selection

**Approach 2 — Adaptive Question Ordering**
- Shuffle คำถามตั้งแต่ต้น (Fisher-Yates)
- หลังตอบแต่ละข้อ: คำนวณ confidence per-dimension = |net| / (answered × 2)
- `pickNextQuestion()`: เลือกข้อถัดไปจาก dimension ที่ confidence ต่ำที่สุด (uncertain-first)
- Early-finish: ถ้า confidence ≥ 0.75 และตอบครบ 3 ข้อทุก dimension → banner "ดูผลเลย"
- Progress indicator: 4 dimension mini-bars แสดง confidence แบบ real-time

### Files changed
- `src/types/mbti.ts` — เปลี่ยน `MBTIAnswer.choice: "A"|"B"` → `likert: 1|2|3|4|5`
- `src/utils/mbti.ts` — rewrite: weighted scoring, DimProgress, pickNextQuestion, canFinishEarly, shuffleArray
- `src/features/mbti/components/mbti-quiz.tsx` — rewrite: Likert UI + adaptive queue + confidence bars + early-finish banner

## — MBTI UX Overhaul: Adaptive Discovery Experience (ตาม mbti-core.md)

### สิ่งที่ทำ
ปรับ quiz UX ทั้งหมดให้รู้สึกเหมือน "ระบบกำลังค้นหาตัวตนของคุณ" แทน "การกรอกฟอร์ม" เพื่อลด drop-off — อ้างอิง `context/features/mbti-core.md`

**1. Minimal Progress (ลบ completion anxiety)**
- ลบ "ข้อ X / 28" และ dots 28 จุดล่างออก
- แสดงแค่ "ข้อที่ N" + dimension ที่กำลัง probe

**2. Confidence-Based Progress Bar**
- progress bar = `overallConfidence()` (ค่าเฉลี่ย dimension certainty) ไม่ใช่จำนวนข้อ
- label "กำลังวิเคราะห์บุคลิก… ความมั่นใจ X%"

**3. Live Personality Preview ⭐ (core curiosity loop)**
- แสดง 4 ตัวอักษรแบบ `I N ? J` — opacity/สีตาม confidence ของแต่ละ dimension
- ตัวที่ยังไม่ชัดเป็น "?" สีเทา, ตัวที่ solid (≥0.75) เป็นสีเขียว + ✓
- มี mini confidence bar ใต้แต่ละตัว

**4. Dimension Completion Celebration**
- dim ไหน confidence ข้าม 0.75 → toast "เข้าใจ{ด้าน}ของคุณแล้ว" (fade+slide, auto-clear 1.9s)
- ตรวจการข้าม threshold ด้วย `newlyCompletedDimensions(prev, next)`

**5. Auto-Finish (ไม่ถาม ไม่มี banner)**
- ลบ early-finish banner ออก
- พอ `canFinishEarly()` = true หลังตอบข้อนั้น → เข้า reveal อัตโนมัติทันที

**6. Adaptive Question Framing**
- บรรทัดบนคำถาม: "ข้อที่ N · ทำความเข้าใจ{ด้าน}ของคุณ" + จุดสีตาม dimension

**8. Cinematic Result Reveal**
- สร้าง `MBTIReveal` component ใหม่: "กำลังสรุป…" → เผยทีละตัวอักษร (พร้อม Thai label) → merge เป็น type → ส่งต่อ result card
- timeline ~4.5s ด้วย setTimeout chain, ใช้ ref กัน effect re-trigger

### Helper functions เพิ่มใน utils/mbti.ts
- `DIM_ORDER` — canonical order
- `getPartialType(progress)` → LetterPreview[] (letter, confidence, solid)
- `overallConfidence(progress)` → 0–1
- `newlyCompletedDimensions(prev, next)` → dims ที่เพิ่งข้าม threshold

### Files changed
- `src/utils/mbti.ts` — เพิ่ม DIM_ORDER, getPartialType, overallConfidence, newlyCompletedDimensions, LetterPreview type
- `src/features/mbti/components/mbti-quiz.tsx` — rewrite UX: live preview hero, confidence progress, celebration toast, auto-finish, adaptive framing (ลบ dots/counter/early-finish banner)
- `src/features/mbti/components/mbti-reveal.tsx` — **(ใหม่)** cinematic letter-by-letter reveal

## — Fix Vercel Build: Prisma Client ไม่ถูก generate

### ปัญหา
Vercel build fail: `Module not found: Can't resolve '@/generated/prisma/client'` ที่ `src/lib/prisma.ts:1`

### สาเหตุ
- `schema.prisma` generate client ไปที่ `src/generated/prisma` (custom output)
- `.gitignore` ignore `/src/generated/prisma` → ไม่ commit ขึ้น git
- `package.json` build script มีแค่ `next build` ไม่มี `prisma generate`
- ผลคือบน Vercel ไม่มีไฟล์ generated และไม่ได้สั่ง generate → หา module ไม่เจอ

### สิ่งที่ทำ
- เปลี่ยน build script เป็น `prisma generate && next build`
- เพิ่ม `postinstall: prisma generate` เผื่อ Vercel dependency cache
- ยืนยัน `prisma generate` รันผ่าน (Prisma Client 7.8.0)
- ตรวจ `prisma.config.ts` — โหลด `.env.local` แต่ Vercel ใช้ env ของตัวเอง + generate ไม่ต้องต่อ DB จึงไม่กระทบ

### Files changed
- `package.json` — `build: prisma generate && next build`, เพิ่ม `postinstall: prisma generate`
