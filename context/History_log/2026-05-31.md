### Phase 3 — เพิ่ม `programCode` เป็น Identity Key บน Faculty

#### สิ่งที่ทำ
- เพิ่ม `programCode String?` และ `@@index([programCode])` ใน `prisma/schema.prisma` (Faculty model)
- รัน `prisma db push` → sync schema สำเร็จ
- แก้ `scripts/import-tcas.ts`:
  - `upsertFaculty()` เพิ่ม `programCode` write field
  - เพิ่ม `preloadFacultyIndex()` — โหลด Faculty+TcasScore ทั้งหมดเข้า in-memory cache ที่ startup เพื่อกัน per-row DB query (ป้องกัน connection timeout)
  - `lookupCache()` — lookup ด้วย `(universityId, programCode, normMajor)` แทน slug
  - cache keys: `${uniId}:${programCode}:${normMajor}` และ `${uniId}:slug:${slug}` (fallback)
- แยก shared normalizers ออกเป็น `src/lib/normalize-faculty.ts`:
  - `normalizeDetail()` แก้ bug: เดิม return `""` เมื่อ detail ขึ้นต้นด้วยชื่อคณะ → ตอนนี้ extract keyword ภาคปกติ/โครงการพิเศษ/หลักสูตรนานาชาติ ออกมาแทน
  - ย้าย `normalizeProgram()`, `normalizeMajor()` จาก queries.ts ไปไว้ที่นี่
- แก้ `scripts/dedupe-faculties.ts` Phase 1 ให้ group ด้วย `normalizeDetail(f.detail, f.name)` แทน raw string

#### Files changed
- `prisma/schema.prisma` — เพิ่ม `programCode String?` + `@@index([programCode])` ใน Faculty

### Phase 5 — COTMES Fix (กลุ่มสถาบันแพทยศาสตร์)

#### ปัญหา
COTMES (กลุ่มสถาบันแพทยศาสตร์แห่งประเทศไทย) **reassign programCode ให้มหาวิทยาลัยสมาชิกต่างกันทุกปี** เช่น:
- TCAS65: `50310123` = ทันตแพทย์ มหิดล
- TCAS66: `50310123` = ทันตแพทย์ จุฬา

ทำให้ exact-code lookup จับ Faculty ผิด (จุฬา TCAS66 ไปชน มหิดล TCAS65)

#### สิ่งที่ทำ
- เพิ่ม `isCOTMES(universityName: string): boolean` → detect ด้วย `universityName.includes("กลุ่มสถาบัน")`
- แก้ `lookupCache()` ใน import-tcas.ts ให้ branch แยกกัน:
  - **COTMES**: ข้าม programCode lookup ทั้งหมด → ใช้ `facultyName`-based key แทน (`${uniId}:fac:${facultyName}:${normMajor}`) เพราะ COTMES faculty name มีชื่อมหาลัยอยู่ใน string เช่น "คณะทันตแพทยศาสตร์ จุฬาลงกรณ์มหาวิทยาลัย"
  - **Regular uni**: ใช้ programCode-based key ตามเดิม
- แก้ `preloadFacultyIndex()` ให้ใช้ COTMES split เดียวกันตอน load existing data
- สร้าง `scripts/debug-cotmes.ts` — debug COTMES Faculty rows ก่อน/หลัง fix
- Truncate DB อีกรอบ → re-import ด้วย logic ใหม่
